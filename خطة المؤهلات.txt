خطة إدارة المؤهلات العلمية للموظفين

1. تصميم قاعدة البيانات (Database Design)

EducationLevels: جدول المستويات العلمية

Id

Name (تعليم أساسي، متوسط، عالي، ماجستير، دكتوراه)

Qualifications: جدول المؤهلات

Id

Name (بكالوريوس تجارة، دبلوم صناعي، إلخ)

EducationLevelId (FK → EducationLevels)

Specializations: جدول التخصصات (اختياري)

Id

Name (محاسبة، كهرباء، إدارة أعمال)

QualificationSpecializations: جدول ربط بين المؤهل والتخصص

Id

QualificationId (FK → Qualifications)

SpecializationId (FK → Specializations)

Employees: جدول الموظفين

EmployeeId

Basic Info (اسم، رقم وظيفي، إلخ)

EmployeeQualifications: جدول مؤهلات الموظف

Id

EmployeeId (FK → Employees)

QualificationId (FK → Qualifications)

SpecializationId (FK → Specializations, Nullable)

IsDefault (Boolean لتحديد المؤهل الأساسي المرتبط بالوظيفة)

Notes (اختياري)

2. المنطق الإداري (Business Logic)

إضافة/تعديل/حذف المستويات، المؤهلات، التخصصات.

إدارة الموظف:

اختيار المستوى العلمي.

اختيار المؤهل المرتبط بالمستوى.

اختيار التخصص (إن وجد).

تحديد المؤهل الأساسي (IsDefault).

إضافة مؤهلات إضافية غير مرتبطة بالوظيفة (ماجستير، دكتوراه).

3. تصميم واجهة المستخدم (UI Design)

شاشة الموظف الرئيسية

بيانات أساسية.

عرض المؤهل الأساسي فقط.

شاشة فرعية (Tab المؤهلات)

جدول يعرض كل مؤهلات الموظف.

زر (Add Qualification).

Checkbox لتحديد المؤهل الافتراضي.

دعم مؤهلات بدون تخصص (Specialization = NULL).

شاشة إدارة المؤهلات العامة

تبويب المستويات العلمية.

تبويب المؤهلات.

تبويب التخصصات.

دعم ربط المؤهل بالتخصصات عبر جدول وسيط.

4. التقارير والاستعلامات (Reports)

استخراج الموظفين حسب المستوى العلمي (مثلاً: دكتوراه).

استخراج الموظفين حسب المؤهل الأساسي المرتبط بالوظيفة.

استخراج الموظفين حسب المؤهلات الإضافية.

5. خطة التنفيذ خطوة بخطوة

بناء الجداول في EF Core مع العلاقات.

إنشاء DTOs و Services للفصل بين الكيانات والعرض.

تصميم صفحات Blazor:

صفحة إدارة المستويات والمؤهلات والتخصصات.

صفحة الموظف + شاشة فرعية للمؤهلات.

إضافة Validation و UX بسيط (Bootstrap Forms + Modals).

اختبار عملي بإدخال بيانات تجريبية.

تحسين الواجهة (ألوان متناسقة، تجنب تكرار الكاردات).

6. المزايا

مرونة في إدارة المؤهلات.

دعم مؤهلات عامة بدون تخصص.

إمكانية تحديد مؤهل افتراضي مرتبط بالوظيفة.

حفظ المسار العلمي الكامل للموظف.

7. الخلاصة

النظام المقترح يوفر إدارة شاملة للمؤهلات العلمية، مع دعم الحالات الخاصة (مؤهلات عامة، مؤهلات إضافية غير مرتبطة بالوظيفة)، ويضمن سهولة الاستخدام والتوسع مستقبلاً.
